# -------------------------
# Conventional Commit Template
# -------------------------

# Commit type (required):
# feat     → new feature
# fix      → bug fix
# chore    → build / tooling / other tasks
# docs     → documentation changes
# style    → formatting / CSS / UI tweaks (no logic change)
# refactor → code changes that neither fix a bug nor add a feature
# test     → adding or updating tests
# perf     → performance improvements
# ci       → CI/CD, pipelines

# Scope (optional, e.g., component or file name):
# (example: header, tailwind, api, App.tsx)

# Short description (required, <= 72 characters)
# -------------------------
# Example:
# feat(header): add responsive nav bar

# Longer description (optional, wrap at ~80 chars)
# - What was changed
# - Why it was changed

# BREAKING CHANGE: (optional)
# - Describe breaking changes here
